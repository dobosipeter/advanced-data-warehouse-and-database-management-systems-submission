\section{System Architecture}

The system implements an end-to-end data warehouse pipeline: ingesting real-time air quality
data from OpenAQ, staging it in raw JSON form, loading it into a normalized OLTP schema,
then transforming it into an analytical star schema in the \texttt{dw} schema.
A machine learning model provides PM2.5 predictions stored back into the warehouse.

\begin{figure}[H]
\centering
\includegraphics[width=\textwidth]{../../diagrams/data_flow.png}
\caption{End-to-end data flow: ingestion → staging → OLTP → ETL → Data Warehouse → analytics.}
\label{fig:dataflow}
\end{figure}

\noindent Key data warehouse components:
\begin{itemize}[nosep]
  \item \textbf{Staging layer} (\texttt{staging} schema) --- raw JSON API responses preserved for auditability.
  \item \textbf{OLTP layer} (\texttt{oltp} schema) --- normalized operational tables with triggers and constraints.
  \item \textbf{Data Warehouse} (\texttt{dw} schema) --- star schema with facts and dimensions.
  \item \textbf{ETL pipeline} --- Python script performing incremental extraction, SCD Type~2 dimension updates, and fact loading.
  \item \textbf{ML prediction} --- Random Forest model trained on warehouse data, predictions stored as \texttt{fact\_prediction}.
  \item \textbf{Analytical dashboard} --- Streamlit multi-page application with filters, trend charts, heatmaps, and prediction comparisons.
\end{itemize}

\section{Functional Specification}

\begin{enumerate}[nosep]
  \item \textbf{Data Ingestion} --- Incremental and bulk ingest from OpenAQ v3, with run logging and error tracking.
  \item \textbf{ETL Pipeline} --- Extracts from OLTP, applies SCD Type~2 logic to location and sensor dimensions, loads facts.
  \item \textbf{Star Schema} --- Three fact tables (measurements, alerts, predictions) with six dimension tables.
  \item \textbf{SCD Type~2} --- Location and sensor dimensions track history with \texttt{valid\_from/valid\_to/is\_current} versioning.
  \item \textbf{Risk Classification} --- PM2.5 values classified into Low / Moderate / High / Critical via \texttt{dim\_risk\_class}.
  \item \textbf{PM2.5 Prediction} --- Random Forest model trained on historical DW data; predictions stored in \texttt{fact\_prediction}.
  \item \textbf{Automated Refresh} --- Orchestration script with modes (full, ingest-etl, predict-only, train-predict) and cron scheduling.
  \item \textbf{Analytical Dashboard} --- Four Streamlit pages: Air Quality Overview, High-Risk Periods, Prediction Insights, Data Operations.
  \item \textbf{Threshold Alerts} --- Configurable alert rules; alerts generated automatically and integrated into the DW as \texttt{fact\_pollution\_alert}.
  \item \textbf{Deployment} --- Docker Compose on Hetzner Cloud VM with Caddy HTTPS, CI/CD via GitHub Actions.
\end{enumerate}

\subsection*{Notable Data Warehouse Features Demonstrated}

\begin{itemize}[nosep]
  \item \textbf{Star schema} with conformed date/time dimensions
  \item \textbf{SCD Type~2} with hash-based change detection for location and sensor dimensions
  \item \textbf{Incremental ETL} with watermark-based extraction (only new OLTP rows loaded)
  \item \textbf{Risk classification dimension} linking measurements to severity bands
  \item \textbf{ML integration} --- predictions stored as facts alongside actuals for comparison
  \item \textbf{Analytical dashboard} with daily aggregation, heatmaps, and error analysis
  \item \textbf{Automated pipeline orchestration} with scheduled cron jobs
\end{itemize}
